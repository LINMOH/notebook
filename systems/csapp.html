Chapter 1: A Tour of Computer Systems

本章梳理计算机系统的核心概念，后续章节将深入探讨。

计算机系统定义

计算机系统 = 硬件 + 软件，用于运行应用程序。

Information Is Bits + Context

  • 系统中所有信息（代码、数据等）都以比特表示。
  • 由 ASCII 字符构成的人类可读文件称为文本文件，其他文件称为二进制文件。

The Compilation System

运行 C 语言程序必须将代码转换成机器语言指令，打包成可执行文件。编译流程如下：

  1. 预处理阶段（Preprocessing Phase） 预处理器（cpp）根据 # 指令修改 C 代码，生成扩展名 .i 的中间文件。

  2. 编译阶段（Compilation Phase） 编译器（cc1）将 .i 文件翻译成汇编语言文件 .s。

  3. 汇编阶段（Assembly Phase） 汇编器（as）将 .s 文件转换为机器代码，生成目标文件 .o。

  4. 链接阶段（Linking Phase） 链接器将目标文件和库文件合并，生成最终的可执行文件。

了解编译系统的意义：

  • 优化程序性能
  • 理解链接错误
  • 避免安全漏洞

Hardware（硬件）

  • 总线（Buses）：系统内部传递信息的电子管道。
  • 字（Word）：固定长度的数据块，如 32 位为 4 字节，64 位为 8 字节。

  • I/O 设备（Input/Output Devices）：连接系统与外部设备，通过控制器或适配器连接到总线。

    • 控制器：芯片组
    • 适配器：插在主板上的卡
  • 主存（Main Memory）：存放程序和数据的临时存储，由 DRAM 组成。
  • 处理器（CPU）：执行存储在主存中的指令，核心是程序计数器（PC），负责指令读取和执行。

执行程序时的流程：

  1. 读取命令

  2. 利用直接存储器存取（DMA）技术，从硬盘直接传输数据到主存

  3. 执行程序，输出数据

OS（操作系统）

操作系统是硬件和软件之间的中间层，提供硬件管理和软件接口：

操作系统基本功能：

  1. 防止应用程序滥用硬件
  2. 提供一致简单的机制控制硬件

抽象概念：

  • 进程：系统资源（CPU、内存、I/O）抽象，可以并发运行。多核处理器可同时执行多个进程。上下文切换是操作系统切换进程的机制，由内核实现。

  • 线程：进程中的执行单元，线程间共享数据，多线程能提升程序运行速度。

  • 虚拟内存：为主存和 I/O 设备提供抽象，每个进程拥有独立的虚拟地址空间：

    虚拟地址空间包含：程序代码、数据段、堆、共享库、栈、内核虚拟内存等。

  • 文件：字节序列，是 I/O 设备的抽象，为应用程序提供统一访问接口。
