    Chapter 1: A Tour of Computer Systems link

    本章是对计算机系统主干部分的梳理，后续章节的内容就是对以下概念的挖掘和深入。

    计算机系统 = 硬件 + 软件，用来运行应用程序。

    Information Is Bits + Context link

    系统中所有信息（代码、数据等）均以比特表示。

    只由 ASCII 字符构成的人类可读文件称为文本文件，除文本文件外的所有其他文件都称为二进制文件。

    The Compilation System link

    在系统上运行 C 语言程序，必须将每条 C 语句转化为一系列的低级机器语言指令，然后按照可执行目标程序的格式打包，以二进制存放，生成可执行目标文件。这一过程为编译系统，如下图：

    1. 预处理阶段 Preprocessing Phase：预处理器（cpp）根据以 # 开头的命令，修改 C 程序，得到另一个 C 程序，以 .i 作为文件拓展名。
    2. 编译阶段 Compilation Phase：编译器（ccl）将文本文件 .i 翻译成文本文件 .s，包含一个汇编语言程序。
    3. 汇编阶段 Assembly Phase：汇编器（as）将 .s 文件翻译成机器语言指令，打包成可重定位目标程序，保存为文件拓展名为 .o 的二进制文件。
    4. 链接阶段 Linking Phase：将库函数和合并到 .o 的文件，这是可执行目标文件，可以被加载到内存中，由系统执行。

    了解编译系统的好处：

    • 优化程序性能
    • 理解链接时出现的错误
    • 避免安全漏洞

    Hardware link

    • 总线 Buses：贯穿整个系统的电子管道，携带信息字节并在各个部件间传递。
      • 字 Word：定长的字节块，是一个基本的系统参数，32 位是 4 字节，64 位是 8 字节。
    • I/O 设备 I/O Devices：链接系统与外部设备，分为输入（Input）和输出（Output）。每个 I/O 设备都通过一个控制器或适配器与 I/O 总线连接。
      • 控制器：芯片组
      • 适配器：插在主板上的卡
        • 主存 Main Memory：临时储存设备，用来存放程序及其数据，由动态随机存取储存器（DRAM）组成，是一个线性的字节数组，每一个字节都有唯一的数组索引。
    • 处理器 Central Processing Unit, CPU：解释或执行主存中的指令，核心是程序计数器（PC），一个字的寄存器，PC 总是含有主存中某条机器语言指令的地址。处理器是按照一个指令集架构的模型来操作的，处理器从 PC 指向的内存处读取指令，执行指令后更新 PC，指向下一条指令。以下是一些简单操作：加载、存储、操作、跳转，围绕储存、寄存器文件、ALU 进行。

    OS link

    操作系统是介于硬件与软件之间的一层软件。

    所有应用程序访问硬件都必须经过操作系统，操作系统由两个基本功能：

    1. 防止硬件被指控的应用程序滥用
    2. 向应用程序提供简单一致的机制控制复杂的低级硬件设备

    为了实现这些功能，操作系统提供了几个抽象概念：

    • 进程：对处理器、内存和 I/O 设备的抽象，可以并发运行，同时运行多个进程。多核处理器同时能够执行多个程序。上下文切换是处理器在进程间切换来并发地执行多个进程的机制。上下文是操作系统保持跟踪进程运行所需的状态信息，当操作系统从当前进程转移到另一进程时，就会发生上下文切换，进程之间的转变通过内核实现，内核是系统管理全部进程所需代码和数据结构的集合。

    • 线程：组成进程的执行单元，线程之间更容易共享数据，多线程也是一种让程序运行的更快的方法。
    • 虚拟内存：主存和 I/O 设备的抽象。虚拟地址空间是每个进程都独立的使用内存。以下是进程的虚拟地址空间：

    • 程序代码和数据
    • 堆
    • 共享库
    • 栈
    • 内核虚拟内存
    • （这里不理解）

    • 文件：字节序列，是 I/O 设备的抽象，向应用程序提供了统一的视图，来看待系统中的 I/O 设备
